Difference between HTML and HTML Email

HTML Emails are HTML tables that are being read in a rich text format within an email client’s rendering engine. The definition of HTML states:

“Hypertext Markup Language (HTML) is the standard markup language for documents designed to be displayed in a web browser.”

HTML has a set of rules, standards and methodologies that are overseen and maintained by W3C for web development purposes.

HTML Emails on the other hand do not have a set rules to follow, but instead email rendering clients have their own way of displaying an email;

  • Apple, iOS Mail, Outlook MAC, Android Mail - Webkit
  • Outlook 2000-2003 - Internet Explorer (Trident)
  • Outlook 2007/2010/2013 - MS Word
  • Mozilla Thunderbird - Mozilla Geko
  • Gmail and Yahoo - Based on the browser used

Because of this, email client rendering engines display HTML very differently to web browsers (especially modern browsers). Email Developers need to create emails as if developing back in the 90s, using tables, deprecated HTML tags, embedded and inline CSS, with MSO tags (for MS Office rendered clients) in order to correctly render and display their emails across different email clients, devices and browsers. Additionally email clients do not allow the use of JavaScript, and restrict the size of the HTML email document and it’s images.

So sending an HTML Email is essentially hacking the rendering client and adding your code in a way for the rendering engine to recognize it, so it serves it up to as rich text.

So you’re asking yourself, ‘Why be an email designer or developer?”. My answer, " It challenges me, and I love it!!!”
